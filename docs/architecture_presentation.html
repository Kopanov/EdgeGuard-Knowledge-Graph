🎯 Sector Classification: Keyword-Based Approach

How We Classify Threats by Zone/Sector

Approach: We use regex word-boundary matching to classify indicators into sectors (zones). This prevents false positives like "phishing" matching within "hospital".

Sector Primary Keywords Threat Context Keywords Current %
Healthcare hospital, medical, patient, pharma, clinic healthcare sector, medical device, covid ~61%
Energy scada, grid, power, industrial, ics energy sector, power grid, ot ~28%
Finance bank, payment, trading, wallet banking trojan, financial theft, crypto ~2%
Global (default) generic threats ~6%

False Positive Prevention: Word boundaries ensure "phishing" doesn't match in "hospital" or "his". We also map malware families (Emotet, TrickBot, QakBot) → Finance sector automatically.

🔗 Neo4j Data Model: Relationship Edge Properties

Relationship Direction Edge Properties Example
USES ThreatActor → Technique confidence, source, timestamp APT29 uses T1059 (0.95)
ATTRIBUTED_TO Malware → ThreatActor confidence, source, timestamp TrickBot → Wizard Spider
INDICATES Indicator → Vulnerability confidence, source, timestamp IP indicates CVE-2024-1234
SOURCED_FROM Any Node → Source raw_data, confidence, source, timestamp Raw data preserved on edge

Design Decision: Raw source data is stored on the SOURCED_FROM edge, not node properties. This keeps nodes clean while preserving all original data for verification and comparison.
